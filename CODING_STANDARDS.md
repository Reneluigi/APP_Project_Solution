GLOBAL CODING STANDARDS (MANDATORY):

LANGUAGE:
- All code must be written in English.
- No Portuguese is allowed in code, variables, comments, or file names.
- UI text can be in English only.

NAMING CONVENTIONS:
- Variables: snake_case
- Functions: snake_case
- Classes: PascalCase
- Constants: UPPER_CASE

EXAMPLES:
- user_company (correct)
- get_user_permissions() (correct)
- CompanyUser (correct)

AVOID:
- usuario_empresa (wrong)
- pegar_dados() (wrong)

CODE STYLE:
- Follow clean code principles
- Avoid abbreviations unless standard
- Use descriptive names

STRUCTURE:
- Organize code for scalability
- Separate concerns (models, views, services, utils)

COMMENTS:
- Only when necessary
- Always in English
- Explain "why", not "what"

DATABASE:
- All table names in lowercase
- Use clear naming:
  - company
  - user_profile
  - project_data

PROFESSIONAL STANDARD:
- Code must be production-ready
- Avoid hacks or temporary fixes
- Always think in scalability

This rule applies to the entire project and must never be violated.
